What is DocuMind AI?

DocuMind AI is a comprehensive, production-grade AI-powered analysis platform designed for modern enterprises that need intelligent document processing and call center quality assurance. The system leverages Google's Gemini 2.0 Flash multimodal AI engine to provide state-of-the-art natural language processing capabilities including document summarization, named entity recognition (NER), sentiment analysis, and audio transcription with compliance scoring.

The platform is built specifically for call center environments where regulatory compliance, quality assurance, and customer interaction analysis are critical business requirements. It processes both text-based documents (PDFs, DOCX files, images) and audio recordings (MP3, WAV, M4A, FLAC, OGG) through a unified API interface, delivering structured JSON outputs suitable for integration with existing enterprise systems, dashboards, and reporting tools.

Document Analysis Pipeline

The document analysis module accepts files in multiple formats and processes them through a multi-stage AI pipeline. When a PDF is uploaded, the system uses pdfplumber for layout-aware text extraction that preserves document structure, tables, and formatting. DOCX files are processed using python-docx to extract paragraphs, headings, and embedded content. Image files (PNG, JPG, JPEG) are sent directly to Google Gemini's multimodal Vision API for OCR and content understanding.

Once text is extracted, it undergoes cleaning and normalization through the text processing module, which removes noise, normalizes whitespace, and prepares the content for AI analysis. The cleaned text is then sent to Gemini 2.0 Flash with a carefully engineered prompt that instructs the AI to produce a comprehensive 20-30 line summary covering all key points, named entities organized by category (people, organizations, dates, monetary amounts), and overall sentiment classification.

For resumes, the system extracts candidate name, contact information, educational background, technical skills, work experience, projects, certifications, and achievements. For reports, it covers executive summaries, methodology, findings, conclusions, and recommendations. For contracts and legal documents, it identifies parties, terms, obligations, and key clauses.

Call Center Compliance Analysis

The Call Center Compliance module is designed to analyze customer service call recordings against Standard Operating Procedure (SOP) requirements. Audio files are uploaded through the frontend, base64-encoded, and sent to the Gemini 2.0 Flash multimodal API which performs speech-to-text transcription and natural language understanding in a single inference pass.

The system evaluates each call against five critical SOP stages: Greeting (did the agent properly greet the customer?), Identification (was customer identity verified?), Problem Statement (was the call purpose clearly explained?), Solution Offering (were solutions or options discussed?), and Closing (was there a proper closing statement?). Each stage is marked as FOLLOWED or NOT_FOLLOWED, and an overall compliance score from 0.0 to 1.0 is calculated.

Beyond SOP validation, the system extracts payment preference intelligence (EMI, Full Payment, Partial Payment, Down Payment), identifies rejection reasons when applicable (High Interest, Budget Constraints, Already Paid, Not Interested), performs sentiment analysis on the overall conversation, and extracts relevant keywords for searchability and reporting.

The system supports multilingual analysis including Tamil (Tanglish), Hindi (Hinglish), and English, making it suitable for diverse Indian call center environments.

Technical Architecture

The backend is built with FastAPI, a modern high-performance Python web framework that supports asynchronous request handling. The API is secured with API key authentication via the x-api-key header. All requests are validated using Pydantic models for type safety and automatic documentation generation. The system runs on Uvicorn ASGI server for production-grade performance.

The AI engine uses Google Gemini 2.0 Flash, which provides multimodal capabilities meaning it can process text, images, and audio in a single model call. This eliminates the need for separate OCR, STT, and NLP services, reducing latency and complexity. The system includes intelligent fallback mechanisms that provide meaningful responses even when the AI API is temporarily unavailable or rate-limited.

Authentication is handled through Firebase Authentication with Google Sign-In support, providing secure and convenient user management. The frontend is a single-page application built with vanilla HTML, CSS, and JavaScript, featuring a modern dark-themed glassmorphism design with smooth animations and responsive layout.

API Endpoints

POST /api/document-analyze Accepts a JSON body with fileName (string), fileType ("pdf", "docx", or "image"), and fileBase64 (base64-encoded file content). Returns a JSON object with summary, entities (names, dates, organizations, amounts), and sentiment. Requires x-api-key header for authentication.

POST /api/call-analytics Accepts a JSON body with language ("Tamil", "Hindi", or "English"), audioFormat ("mp3", "wav", "m4a", "flac", "ogg"), and audioBase64 (base64-encoded audio content). Returns transcript, summary, sop_validation object, analytics object, and keywords array. Requires x-api-key header.

GET /health Simple health check endpoint that returns {"status": "healthy"} when the server is running.
